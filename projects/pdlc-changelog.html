Process Design · PDLC · Iteration

The Pipeline,
Revised.

Every version of the PDLC that ever ran. What it got wrong, what got cut, and what got sharper — phase by phase.

Phase 01 The Build 2025
  • Six agents, five gates: Brainstorm → Intake Clarifier → PRD Draft → Research → PRD Refine → UX Research → Prototype
  • Research was designed to test the PRD — not the raw idea
  • Intake Clarifier ran silently between the idea step and everything else, converting the handoff into a structured brief
  • Agents labeled by what they produce, not by what role they play
  • No defined file structure — outputs written wherever
  • No defined agent names — "PRD Writer," "Research Agent"
  • Pipeline had no kill switch — if research proved the idea wrong, it kept going anyway
Phase 02 The Audit March 2026
  • Ran five gstack skills against the pipeline — the same methodology the PDLC uses on products, turned back on itself
  • Found: no kill switch — research proving an idea wrong didn't stop the plan from being written. Fixed: Gate 1.5 can now route back to brainstorm, not just forward
  • Found: research could agree with everything and nobody would notice. Fixed: Gate 1.5 now flags explicitly if research changed nothing
  • Found: UX conflict loop had no exit — could run indefinitely. Fixed: capped at 3 rounds; human decides if unresolved
  • Found: deepest research tier was quietly skipped when early results came back empty. Fixed: skipping it is now an explicit choice, not a default
  • Found: agents were defined by output, not role — no way to know how to read or push back on what they made. Fixed: six agents named — Structural formatter, Vision writer, Adversarial analyst, Evidence editor, Experience designer, Builder
  • Found: every prototype made its own visual decisions from scratch. Fixed: DESIGN.md captures design decisions once; every future prototype reads it first
Phase 03 The Restructure May 2026 In progress
  • Found: research was running on the PRD draft, not the raw idea — it was testing the plan, not validating the idea. Fixed: research now runs first, before any planning begins
  • Found: PRD Draft + PRD Refine were two agents doing one job in sequence. Fixed: collapsed into a single PRD agent that writes once, already informed by research
  • Found: Intake Clarifier was redundant if both entry agents already produce a structured brief in the same format. Fixed: removed — Brainstorm and Discovery both now output output/brief.md directly
  • Found: no single entry point — users had to know which agent to open. Fixed: added Orchestrator; reads .state.md, routes to current step if resuming, asks one question if starting fresh
  • Defined file structure: output/ for all deliverables, .working/progress/ for intermediate work, .state.md for pipeline state
  • Removed "Gate" terminology throughout — replaced with plain language: "Your call," "Hard lock," "back here"
  • Replaced text-based flowchart with standard Mermaid diagram — diamonds for decisions, color-coded agents, back-loop arrows for all rejection paths

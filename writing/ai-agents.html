Things I learned from building AI agents

  1. 1

    Let AI figure out the gaps. Do not try to dictate every single detail.

  2. 2

    Learn how to build the overall architecture of agents, especially if you are building workflows.

  3. 3

    If you are a non-developer like me, understand the directory route. Know where the inputs and outputs are stored, what reads what, and what should be stored where. Dictate that clearly in the .md files.

  4. 4

    Think about token efficiency because the more you type into prompts, the more tokens you use. But the point is not to write less, it is to write cleaner prompts.

  5. 5

    Clear role, clear input, clear task, clear output format, and clear constraints matter more than long instructions.

  6. 6

    Before letting AI write the entire .md file, start with a flowchart first and tweak it little by little, especially if you are using lighter or cheaper AI tools like Claude, Gen AI, etc.

  7. 7

    Role-play with the agents to make sure you get the desired outputs.

  8. 8

    Once you have a solid .md file, test it with different tools like Claude, OpenAI, and others. Each model has different training, reasoning patterns, context handling, and instruction-following behavior.

  9. 9

    GitHub with Claude Code is a godsend. It reduces so much time, and once it is connected, it is much easier to work with, even for non-developers.
